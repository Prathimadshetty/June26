    SCA Tool and CI Integration¶

    Snyk is a developer-first security solution that helps organizations to use open source and stay secure. Snyk is the only solution that seamlessly and proactively finds and fixes vulnerabilities and license violations in open source dependencies.

    • Snyk statically analyzes software components of the projects to find vulnerable dependencies used and helps to fix them
    • Snyk tests can be integrated with Github and CI/CD tools to trigger scans during pull or build process initiated from developers.

    How to generate API token to access the Snyk CLI ​​​​​​?¶

    Once access is granted to Y account follow the below steps to generate API token. Logon to Snyk UI, using (SSO Login). Go to Organization Settings > Service Accounts > Create a new service account(provide a name) > Provide Role as Org Admin.

    Steps to be followed for performing Snyk CLI scan:¶

    Step 1: Download the Snyk CLI executable (Latest Version) Step 2: Authorize Snyk CLI with Service Account Token.​​​​​​​​​​​​​​ snyk auth

    Step 3: Run snyk test to check for vulnerabilities and fail the build if any are found​​​​​​​ snyk test --all-projects 

             Note: For gradle projects use --all-projects for monorepos or use --all-sub-projects for "multi-project" configurations and test all sub-projects

    Step 4: Capture the dependency tree for ongoing monitoring in Snyk​​​​​​​ snyk monitor --all-projects

    Integration:¶

    Azure Devops¶

    Step 1: Store Service Account token in Azure KeyVault.

    Step 2: Create Azure Resource Manager Service Connection in your Azure Devops Project.

    Step 3: Use below task to get Personal API token from Azure KeyVault.

    task: AzureKeyVault@1
    inputs:
      azureSubscription: '<Service_Connection_Name>'
      KeyVaultName: '<KeyVault_Name>'
      SecretsFilter: '<keyName>'
      RunAsPreJob: true

    Step 4: Use below task to run a snyk test and capture the dependency tree for ongoing monitoring in Snyk.

    Note: Get the Snyk CLI download URL from here based on your OS type.

    task: Bash@3
   displayName: "Snyk_SCA_Scanning"
   inputs:
     targetType: 'inline'
     script: |
       echo "----------Download Latest Snyk CLI-----------"
       curl -Lo ./snyk "https://github.com/snyk/snyk/releases/download/v1.1159.0/snyk-linux"
       chmod +x snyk
       ls -lrt
       echo "----------Authenticating Snyk-----------"
       ./snyk auth $(snyk-token)
       echo "----------Snyk Monitor-----------"
       ./snyk monitor --all-projects
       echo "----------Snyk Scanning-----------"
       ./snyk test --all-projects
     failOnStderr: true
task: Bash@3
   displayName: "Snyk Security Gating"
   inputs:
     targetType: 'inline'
     script: |    
       snykCount=$(curl -X GET --header 'Content-Type: application/json' --header 'Authorization: token $(snyk-token)' 'https://api.snyk.io/api/v1/org/<org id>/project/<project id>')
       CRITICAL=$(echo "$snykCount" | jq .issueCountsBySeverity.critical )
       HIGH=$(echo "$snykCount" | jq .issueCountsBySeverity.high )
       MEDIUM=$(echo "$snykCount" | jq .issueCountsBySeverity.medium )
       LOW=$(echo "$snykCount" | jq .issueCountsBySeverity.low )
       echo "CRITICAL = $CRITICAL"
       echo "HIGH = $HIGH"
       echo "MEDIUM = $MEDIUM"
       echo "LOW = $LOW"
       if [ $CRITICAL -gt 0 ]
       then
         curl -H 'Content-Type: application/json' -d '{"text": "<Application Name> has Critical Issues in Snyk"}' $(teams_webhook)          
         echo "##[error]Application has critical issues"
         #exit 1
       elif [ $HIGH -gt 0 ]
       then
         curl -H 'Content-Type: application/json' -d '{"text": "<Application Name> has High Issues in Snyk"}' $(teams_webhook)          
         echo "##[error]Application has High issues"
         #exit 1
       else
         echo "PASSED"
       fi


Note: To enforce security gating in block mode please uncomment exit code 1.

    GitHub Actions¶

    Step 1: Create Service Account Token as GitHub Secrets in GitHub Settings > Secrets > Repository Secrets.

    Step 2: Use below pipeline to run a snyk test and capture the dependency tree for ongoing monitoring in Snyk.

    Note: Get the Snyk CLI download URL from here based on your runner machine OS type

    name: Snyk_SCA_Scan
on: [push]
jobs:
  Snyk_SCA_Scan:
    needs: Code_checkout
    runs-on: <runner-machine>   
    steps:
    - uses: actions/checkout@v1
    - name: Snyk SCA Scanning
      continue-on-error: true
      run: |
        echo "----------Download Latest Snyk CLI-----------"
        curl -Lo ./snyk "https://github.com/snyk/snyk/releases/download/v1.1159.0/snyk-linux"
        chmod +x snyk
        ls -lrt
        echo "----------Authenticating Snyk-----------"
        ./snyk auth ${{secrets.Snyk_token}}
        echo "----------Snyk Monitor-----------"
        ./snyk monitor --all-projects
        echo "----------Snyk Scanning-----------"
        ./snyk test --all-projects
        echo "-----------Snyk Security Gating-----------"
        snykCount=$(curl -X GET --header 'Content-Type: application/json' --header 'Authorization: token ${{secrets.Snyk_token}}' 'https://api.snyk.io/api/v1/org/<org id>/project/<project id>')
        CRITICAL=$(echo "$snykCount" | jq .issueCountsBySeverity.critical )
        HIGH=$(echo "$snykCount" | jq .issueCountsBySeverity.high )
        MEDIUM=$(echo "$snykCount" | jq .issueCountsBySeverity.medium )
        LOW=$(echo "$snykCount" | jq .issueCountsBySeverity.low )
        echo "CRITICAL = $CRITICAL"
        echo "HIGH = $HIGH"
        echo "MEDIUM = $MEDIUM"
        echo "LOW = $LOW"
        if [ $CRITICAL -gt 0 ]
        then
          curl -H 'Content-Type: application/json' -d '{"text": "Application has Critical Issues in Snyk"}' ${{secrets.TEAMS_WEBHOOK}}
          echo "##[error]Application has critical issues"
          #exit 1
        elif [ $HIGH -gt 0 ]
        then
          curl -H 'Content-Type: application/json' -d '{"text": "Application has High Issues in Snyk"}' ${{secrets.TEAMS_WEBHOOK}}
          echo "##[error]Application has High issues"
          #exit 1
        else
          echo "PASSED"
        fi

Note: To enforce security gating in block mode please uncomment exit code 1.

    Jenkins¶

    Step 1: Store Service Account token in Azure KeyVault.

    Step 2: Create a service principal and store that as a jenkins secret to retrieve the secret from the azure keyvault

    Step 3: Create a Jenkins pipeline with the below stages.

  agent any
  stages {
    stage('Git Checkout'){
            steps {
                checkout([$class: 'GitSCM', branches: [[name: '<branch_name>']], doGenerateSubmoduleConfigurations: false, extensions: [], submoduleCfg: [], userRemoteConfigs: [[credentialsId: '<Github_credential_name>', url: '<Github_url>']]])
                }
    }
   stage ("Retrieving snyk secret from the Azure key Vault") {
     options{
                azureKeyVault(
​
                    credentialID: '<AzureKuyVault_credential_name>',
​
                    keyVaultURL: '<KeyVault_url>',
​
                    secrets: [
​
                        [envVariable: 'snyktoken', name: '<AKV_snyk_token_name>', secretType: 'Secret']
​
                        ]
                        )
     }
      steps {
​
          sh """
                  set -eu -o pipefail
                  rm -rf snyk

                  curl -Lo ./snyk "https://github.com/snyk/snyk/releases/download/v1.1159.0/snyk-linux"

                  ls -lrt

                  chmod +x snyk

                  ./snyk auth ${snyktoken}

    Concourse CI¶

    Step 1: GitHub and Snyk details should be updated in the Hashicorp Vault. (Don't hard code username and password values in the pipeline)

    Use below variables to pass secrets in the pipeline.

    github.username - Github Username github.personal_access_token - Github PAT Token ​​​​​​​snyk_cli.token - Snyk's API token All the variable fields are configured in Hashicorp vault for reusability and manage the secrets ((Hashicorp vault variable)).

    For steps to create a secret keys in HC Vault, refer here.

    Step 2: Create a pipeline.yml with following code

    resources:
  - name: source-code
    type: git
    source:
      uri: <Github_Url>
      branch: <branch_name>
      username: ((github.username))
      password: ((github.personal_access_token))
​
jobs:
  - name: source-code
    public: false
    plan:
      - get: source-code
        trigger: true
      - task: Snyk_SCA_Scanning
        config:
          platform: linux
          image_resource:
            type: registry-image
            source: { repository: ubuntu, tag: "18.04" }
          inputs:
            - name: source-code
          run:
            path: /bin/sh
            args:
              - -c
              - |
                echo "----------Installing Dependency - Curl-----------"
                apt-get update
                apt-get -y install curl
                curl --version
                echo "----------Downloading Snyk-----------"
                curl -Lo ./snyk "https://github.com/snyk/snyk/releases/download/v1.1159.0/snyk-linux"
                chmod +x snyk
                ls -lrt
                echo "----------Authenticating Snyk-----------"
                ./snyk auth ((snyk_cli.token))
                echo "----------Snyk Monitor-----------"
                ./snyk monitor --all-projects
                status=$?
                echo "----------Snyk Scanning-----------"
                ./snyk test --all-projects

    Product Features¶

    • Snyk tool creates inventory of all open-source components used.

    • Snyk tool identifies the known vulnerabilities in the open-source components used.

    • Snyk tool helps in remediating the vulnerable components providing fix recommendations.

    • Find and fix vulnerabilities in application code in real-time during the development process.​​​​​​​

    • Application security at scale requires developers to be the first step in the security process. Snyk’s platform is purpose-built to be easily used by developers to build software securely.
    • The Snyk Intel vulnerability database is maintained by a dedicated research team that combines public sources, contributions from the developer community and academia, proprietary research and machine learning.
    • Snyk will integrated with major repositories like GitHub and major CICD tools like Azure Devops, Jenkins, Concourse, etc...
