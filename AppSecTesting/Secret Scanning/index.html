    Secret Scan

    Secret scanning

    We can use TruffleHog in CI to prevent secrets from being merged in Git. This document explains how to integrate Trufflehog in DevOps pipeline in CI.

    At M&S we use Trufflehog for scanning the repos for finding hardcoded secrets.

    How to integrate?

    General approach : Platform independent CI integration-

    Option 1: Trufflehog Binary(Enterprise Scan)(Recommended)

    You can run Trufflehog Enterprise Scan with Trufflehog Binary using the below command-

    "curl -q "https://storage.googleapis.com/thog-releases/trufflehog-scanner/latest/fetch.sh" | bash
            ./trufflehog --fail-verified git"

    curl command will fetch the Trufflehog scanner latest version for installation

    ./trufflehog â€“ does the actual scanning

    --fail-verified- Only emit failure code for verified findings.

    TruffleHog will return a non-zero exit code if there are findings.

    Azure DevOps pipeline Integration

    trigger:
- 'develop'



pool:
  vmImage: ubuntu-latest

steps:
  - task: Bash@3
    inputs:
      targetType: 'inline'
      script: |
        curl -q "https://storage.googleapis.com/thog-releases/trufflehog-scanner/latest/fetch.sh" | bash
                ./trufflehog --fail-verified git
                 status=$?
                  echo $status
                 if [ $status -eq 1 ]
                then
                 curl -H 'Content-Type: application/json' -d ' {"text":  "Trufflehog Scan for the (ApplicationName) has secrets"  }' $(WEBHOOK)
                else
                 echo "Your application has no secrets"
                fi

Note : In the place of (ApplicationName) in the curl command Mention the Application name which is to be integrated

    Github Actions pipeline Integration-

    steps:
    - uses: actions/checkout@v1
    - name: trufflehog scanning
      continue-on-error: false
      run: |
        curl -q "https://storage.googleapis.com/thog-releases/trufflehog-scanner/latest/fetch.sh" | bash
        ./trufflehog --fail-verified git
         if: always()
    - name: Notify dedicated teams channel for secrets
      if: failure() 
      run: |
         curl -H 'Content-Type: application/json' -d ' {"text": "Your application has secrets" }'  ${{ secrets.webhook}}
         echo "Secrets found"
      continue-on-error: true 
    - name: Notify dedicated teams channel for secrets not found
      if: success()
      run: |
            echo "No Secrets"
            exit 0
       continue-on-error: true

Note : In the place of (ApplicationName) in the curl command Mention the Application name which is to be integrated

Note - Another approach is to use the TruffleHog Enterprise Action which is recommended.

    name: TruffleHog Enterprise scan

    on: [push, pull_request]

jobs:
  scanning:
    name: TruffleHog Enterprise scan
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v2
        with:
          fetch-depth: 0 # fetch all history so multiple commits can be scanned
      - name: TruffleHog Enterprise scan
        uses: trufflesecurity/TruffleHog-Enterprise-Github-Action@main
        with:
            args: --fail-verified ${{ github.event.repository.default_branch }} HEAD

Note : In the place of (ApplicationName) in the curl command Mention the Application name which is to be integrated
